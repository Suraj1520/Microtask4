Microtask 2

The API is used to calculate the visitor count of website and pagwview count of the website which is using a local database in count.json. The authentication system is designed for signin and signup with error handling. The authentication system uses json web token and hashing of password with salt to provide reliable security.
The database used is MongoDB
The API is live on deployment server i.e. Deta.sh : https://ka40wq.deta.dev/

Description

The main function of the api made for this task are:

  • Authentication: For Authentication I have created two checkpoints i.e either the user is new i.e. he/she have to signup or the user have already signup he/she have to sigin.After login successfully since I have connected it to mongodb as it is my database it will generate the { username , password , email } as the result also there will be a token which can be used for checking authentication and can be used for several other purpose in which token is required.
  • Visitor count: I have created an express server and made an array having count of pageview and visitor count which is dynamically updated by a function written in visitor.js file. So when a visitor first run the website pageview=1 && visit=1 but when he/she refreshes the page, pageview increases accordingly but visit remains the same but if he/she opens the URL in new browser then view=2 and increases dynamically and pageview also gets updated.

Endpoints

                                    #For authentication :
                                     Signup : '/users/signup'
                                     Signin : '/users/signin'
                                              
                                    #For visitor count (preffered to run locally):
                                     To increase pageview : '/visitorcount/api?type=pageview'
                                     To increase pageview as well as visit count : '/visitorcount/api?type=visit-pageview'

MicroTask 3

The API gives geolocation of any area by giving longitude and latitude of area
The API is live on deployment server i.e. Deta.sh : https://dk6vhm.deta.dev/

About

The api created under this microtask gives the geolocation of any area using the result fetched by the api https://nominatim.org/release-docs/develop/api/Reverse/

Endpoints

The various endpoint related to this api is: 

                        'api/getAdd?lat=[latitude]&lon=[longitude]'

The above endpoint will give the desired geolocation provided the latitude and longitude are correct
